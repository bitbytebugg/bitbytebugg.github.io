clang和clang-cc的区别

  • clang 是一个 驱动程序（Driver）。它的主要任务是解析你通过命令行传入的参数（例如 -o, -g, -c, -l），然后根据这些参数去调用真正的编译工具链中的其他工具，并组织好调用顺序和参数传递。可以通过clang –help查看其支持的参数是很简洁的，方便直接调用。
  • clang-cc 才是真正的编译器前端，主要用于 Clang 开发者和高级用户进行调试、探究编译器的内部行为，或者需要非常精细地控制前端时使用。通过 clang-cc –help查看其支持的参数更复杂，更底层。
  • 可以通过传入clang的-###参数查看内部调用的 clang-cc命令：

这里clang-cc各个参数的作用：

  • “-triple” “x86_64-pc-cygwin”: 表明体系结构和操作系统的三元组
  • “-emit-llvm”: 生成LLVM IR
  • “-disable-free”: 退出时禁用内存释放，这个可以不设置
  • “-main-file-name” “hello_llvm.c”: 输入的主要文件
  • “–relocation-model” “static”: 使用静态重定位
  • “–disable-fp-elim”: 禁止帧指针frame pointer的优化
  • “–unwind-tables=1”: Generate unwinding tables for all functions
  • “–mcpu=x86-64”：指定cpu架构
  • “–fmath-errno=1”：不知
  • “-fdiagnostics-show-option”：显示错误诊断信息
  • “-o” “hello_llvm.ll”：输出文件
  • “-x” “c”: 编译语言类型为C语言
  • “hello_llvm.c”: 输入文件

/// ParseAST.cpp

void clang::ParseAST() {
    // 后续的Lexing和Parsing出AST的过程都交给这个实例
    Parser P(PP, S);
    P.Initialize();
    Parser::DeclGroupPtrTy ADecl;
    // 分词和解析AST逻辑就在下面这个函数里，ParseTopLevelDecl返回true表示解析到文件EOF，否则返回false
    while (!P.ParseTopLevelDecl(ADecl)) { 
        if (ADecl)
            Consumer->HandleTopLevelDecl(ADecl.getAsVal<DeclGroupRef>());
    };
    Consumer->HandleTranslationUnit(Ctx);
    
}
  • 这里解析逻辑可以配合C99标准文档（其中带锁的是需要付费购买的）食用。也可以参考微软文档里比较简洁的C语言语法介绍External Definitions。
  • 每次调用ParseExternalDeclaration()都在做词法解析的同时也在做语法解析，解析出的AST存放在Context里。
  • ASTConsumer虽然不是抽象类，但是是一个空基类，实现为空，其具体行为肯定是通过其具体派生类来实现的。
    • 不管是什么派生类，需要重点关注HandleTopLevelDecl和HandleTranslationUnit这两个方法的具体实现逻辑。

/// Frontend/Backend.cpp

ASTConsumer *clang::CreateBackendConsumer(BackendAction Action,
                                          Diagnostic &Diags,
                                          const LangOptions &LangOpts,
                                          const CompileOptions &CompileOpts,
                                          const std::string& InFile,
                                          llvm::raw_ostream* OS,
                                          LLVMContext& C) {
  return new BackendConsumer(Action, Diags, LangOpts, CompileOpts,
                             InFile, OS, C);
}

//****************** 【1】重写的HandleTopLevelDecl方法如下 *********************
virtual void HandleTopLevelDecl(DeclGroupRef D) {
  PrettyStackTraceDecl CrashInfo(*D.begin(), SourceLocation(),
								 Context->getSourceManager(),
								 "LLVM IR generation of declaration");
  if (CompileOpts.TimePasses)
	LLVMIRGeneration.startTimer();
  // Gen字段通过函数CreateLLVMCodeGen初始化，返回的类型是CodeGeneratorImpl，最终会调用CodeGenModule::EmitTopLevelDecl函数来处理。
  // 猜测这里就对每个顶层定义进行处理，注意这里是个空函数。
  Gen->HandleTopLevelDecl(D);

  if (CompileOpts.TimePasses)
	LLVMIRGeneration.stopTimer();
}

//****************** 【2】重写的HandleTranslationUnit方法如下  *********************
virtual void HandleTranslationUnit(ASTContext &C) {
  {
	PrettyStackTraceString CrashInfo("Per-file LLVM IR generation");
	if (CompileOpts.TimePasses)
	  LLVMIRGeneration.startTimer();
    // Gen字段通过函数CreateLLVMCodeGen初始化，返回的类型是CodeGeneratorImpl，最终会调用CodeGenModule::Release()来做最后的收尾工作
	Gen->HandleTranslationUnit(C);

	if (CompileOpts.TimePasses)
	  LLVMIRGeneration.stopTimer();
  }

  // 前面是生成LLVM IR，这里开始创建Pass然后生成本地汇编代码
  EmitAssembly();
  
  // Force a flush here in case we never get released.
  if (AsmOutStream)
	FormattedOutStream.flush();
}
