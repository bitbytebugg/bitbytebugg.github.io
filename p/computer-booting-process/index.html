计算机启动过程

Aug 05, 2024
11 minute read

我们启动计算机的第一步实际上是从插上主机电源开始的，因为从此时开始主板上已经产生电信号了。所以我们的故事也从计算机电源的开启过程讲起。

ATX供电电路

PC电源不仅输出电压，还要与主板有信号联系，两者在时间次序上有一定的关系，这个次序也是数字电路中经常提到的概念——时序。时序是电源与主板良好配合的重要条件，也是导致电脑无法正常开关机，以及电源与主板不兼容的最常见原因。

ATX电源和主板在开机过程中交互的信号有两个：

  1. PS_ON#: #表示这个信号是低电平有效的，这个信号通常由主板控制（机箱开关按钮连接的是主板上的IO芯片），电源收到这个信号则会启动电源，提供5V, 3.3V, 12V各路电压。
  2. PWR_GOOD/PWR_OK: 也写作P.G或者P.OK信号。电源收到PS_ON#信号后启动，然后在所有输出电压正常建立后的0.1~0.5秒内，电源将会把P.G信号变成高电平传回给主板，表示电源已经准备好，然后主板开始启动和运行。

ATX电源和主板在关机过程中信号交互过程是：

  1. 主板开机电路驱动PS_ON#恢复为高电平
  2. 电源拉低P.G信号，告诉主板电源马上就要下电，主板立刻进行硬件的紧急复位，以保护硬件不会受损。
  3. 很短的一段时间后电源各路电压关闭，+5VSB的待机电压不会关闭。（所以只要插上电源插头，即使没有打开开机按钮，主板上也是带电的）

设计P.G信号的原因：

微机开机后，由于各路输出电压的时序不同，易对主板的工作状态造成混乱，因此，电源中特设了一个延迟的P.G信号提供给主机板，该信号从IBM－PC诞生时就有，一直沿用至今。对该信号的要求是：开机后，相对于＋5V电压，延迟100ms～300ms产生，在关机时，则提前100ms消失。

电源保持时间：

另一个概念 电源保持时间和 P.G信号的关系不大。电源保持时间是指在失去交流电输入时，PC电源在100%负载的情况下，各路输出的保持时间（因为有电容元件的关系，插头断电后电源输出线上的电流不会立刻归0，一般主电容越大，这个电流持续时间越长）。ATX规范里这项标准要求不低于16ms。当非正常关机时，主板无法给出关机信号，此时电源会探测到交流断电，并把P.G信号变为低电平通知主板，主板立刻进行硬件的紧急复位，以保护硬件不会受损。这种情况电源通知主板断电后，至少还要保持千分之一秒的正常输出电压，供主板进行复位，否则有可能造某些硬件的损坏。我猜测这里保持时间超过1ms并不能保证硬件不损坏，因为上面说过各路电压在P.G信号消失的100ms后断电，所以如果保持时间在100ms以上应该是可以保证硬件不损坏的。但是ATX规范的电源保持时间16ms虽然比1ms大但是小于非断电保持时间100ms，即使达到ATX规范给的断电保持时间，还是有可能出现硬件损坏的。所以电源断电保持时间这个参数并不是严格意义上的断电保护机制，而是一种确保断电保护机制（UPS）能够实行的措施，所以16ms只是给UPS启动并正常工作的时间。

主板触发电路

“主板触发电路”应该是电脑维修中的一个专业的术语。

我觉得这个触发电路示意图画的很好，抽象并简化了芯片之间的链接和交互信号，对于初学者很友好。台式电脑主板触发电路原理-成都匠巢维修技术学校 有了触发电路的知识，我们就可以知道只要插上PSU的插头并打开其开关，主板上就会有+5V的待机信号（专业术语叫做 +5V Stand by，简写为+5VSB）。按下机箱开关会拉低IO芯片的一个引脚，然后IO芯片连接PCH（南桥）的一个引脚同步被拉低，此时PCH检查待机条件，满足就通过同一个引脚发送SLP_S3#信号给IO芯片，这时IO芯片在通过下方红色引脚触发电源的开机信号PWR_ON#，电源开始启动并提供各路电压及向主板发出P.G信号。

主板复位电路

复位电路是主板启动过程中非常关键的部分，确保各个组件能够正确初始化。下面是复位电路的基本工作原理及一些关键点：复位电路工作原理

  1. 复位信号源：
    1. 复位信号产生：主板上的所有复位信号主要由南桥（ICH, I/O Controller Hub）芯片产生。
    2. 复位信号来源：南桥的复位信号来源于ATX电源的灰线（PW_OK），该信号在电源启动时有一个短暂的延迟（通常为100-500毫秒）。
    3. 信号变化：灰线在电源启动瞬间从0V跳变为5V，这个0-1的脉冲信号会直接或间接（通过门电路）作用于南桥，使南桥复位。
  2. 南桥复位
    1. 南桥复位条件：南桥在收到稳定的灰线信号后，内部的复位信号产生电路会将5V电位转换成不同的复位信号，分配给主板上的各个需要复位的设备或模块。
  3. 复位信号分配
    1. 复位信号分配：南桥接收到电源良好信号（PWR_GD）后，会产生一个初始的复位信号PCIRST#，并通过复位门电路逻辑转换为多个复位信号。
    2. PCIRST1#：用于复位板载设备，如I/O芯片、BIOS、北桥、1394芯片等。
    3. PCIRST2#（或DEVRST#、SLOTRST#）：用于复位PCI插槽上的设备。
    4. IDERST#：用于复位IDE设备。
  4. CPU复位
    1. CPURST#产生：当北桥接收到PCIRST1#信号并且其电压及时钟条件满足后，会通过内部电路转换为CPURST#信号来对CPU进行复位。

CPU Reset Vector

关于CPU复位向量网上有两种解答，0xFFFF_FFF0、0xF_FFF0。当然第一种是对的，这个Intel SDM vol3 chapter9有详细说明(9.1.4 First Instruction Executed)就是第一种。这个地址自从intel发布80386后就延续下来了。第二种地址在早年间也是对的，在intel还在使用16位cpu也就是8086和8088时，地址空间是1M，第一条指令的地址就是0xF_FFF0。回答这种地址应该看的是早年的书，或者是理解错了，认为PC刚开机时是16位实模式，这种模式的地址计算方式是使用CS和IP寄存器中的值($CS \times 16 + IP$)，这么算下来确实是0xF_FFF0。但是复位时并不是使用这种计算方式，虽然CPU初始确实是16位实模式，但这是在复位后进入16位模式的。实际上，Intel规定32位、64位CPU复位向量就是0xFFFF_FFF0，只有CS寄存器中初始值被改变后才进入16位模式。

上图第二段话说CS寄存器有两部分：可见的selector部分（存放在CS寄存器里）和不可见的基地址部分。在实模式里，基地址的值是通过将16位selector的值左移4位产生的20位基地址值。然后在硬件复位期间，位于CS寄存器里的可见selector部分加载的值是F000H，而不可见的基地址部分加载的值是FFFF_0000h，此时EIP寄存器的是FFF0h，所以复位时的开始取值地址就是不可见基地址值+EIP的值（即 FFFF_0000h + FFF0h = FFFF_FFF0h）。

上图第三段话说一旦CS寄存器在硬件复位后加载了一个新的值，处理器才会执行实模式的地址计算逻辑（也就是 CS base address = CS segment selector * 16）。所以为了保证在EPROM里的软件初始化代码完全执行之前，代码里都不能包含一个far jump或者far call或者一个中断发生（这些都会导致CS selector的值被改变）。当然这里Intel给的做法并不需要奉为圭臬，我们可以第一条指令就用长跳转（实际BIOS的写法）然后再从16位模式切换到32位或者64位模式。

CPU被复位（复位/使能都是电子技术专业术语）后，CPU开始寻址 0xFFFF_FFF0h （这个地址就被叫做reset vector）。该地址位于4GB地址空间之下，这个地址被永久的映射到在SPI flash memory的一块区域，该区域存储着UEFI的固件。然后CPU从flash memory里开始执行指令然后security phase (UEFI SEC)被初始化。固件在开始的大约20条指令里将CPU切换到64-bit mode（flat protected mode）。Main memory (RAM) 此时还没有被初始化，所以固件会配置CPU caches作为memory使用，此时下一个阶段是 PEI and DXE。注意这些仅仅适用于x86-64平台，在其他系统，例如嵌入式 ARM SoCs，过程会完全不同。ref.

BIOS代码存储在EPROM里，通过硬件也就是北桥南桥芯片把EPROM内的程序映射到两个地方，一个是从FFFF_FFFFh（4GB）处向下扩展，另一个是从F_FFFFh（1MB）处向下扩展）。 第一个映射是因为现代CPU第一条指令在FFFF_FFF0h，所以EPROM必须在那个位置。第二个映射是为兼容16位CPU时代。注意此时内存还无法使用，只是把地址空间分配给了EPROM，CPU访问这段地址就相当于在直接访问EPROM里的数据。这里地址空间受CPU地址线个数控制，不是受RAM的空间限制。

前端总线

Front Side Bus，缩写FSB。早期计算机北桥的MCH和IOH都在CPU外部，都是是通过FSB和CPU连接，所以此时它是CPU和外界交换数据的主要通道（内存，显卡，还有南桥的一些设备）。现代CPU已经摒弃了FSB，采用HT总线或者QPI总线。

HT总线

Hyper-Transport总线，是AMD为K8平台专门设计的高速串行总线。它的发展历史可回溯到1999年，原名为“LDT总线”(Lightning Data Transport，闪电数据传输)。 2001年7月这项技术正式推出，AMD同时将它更名为Hyper-Transport。随后，Broadcom、Cisco、Sun、NVIDIA、ALI、ATI、Apple等许多企业均决定采用这项新型总线技术，而AMD也借此组建Hyper-Transport技术联盟（HTC），从而将Hyper-Transport推向产业界。Hyper-Transport本质是一种为主板上的集成电路互连而设计的端到端总线技术，目的是加快芯片间的数据传输速度。Hyper-Transport技术在AMD平台上使用后，是指AMD CPU到主板芯片之间的连接总线（如果主板芯片组是南北桥架构，则指CPU到北桥），即HT总线，类似于Intel平台中的前端总线（FSB），但Intel平台目前还没采用。在基础原理上，Hyper-Transport与目前的PCI Express非常相似，都是采用点对点的单双工传输线路，引入抗干扰能力强的LVDS信号技术，命令信号、地址信号和数据信号共享一个数据路径，支持DDR双沿触发技术等等，但两者在用途上截然不同—PCI Express作为计算机的系统总线，而Hyper-Transport则被设计为两枚芯片间的连接，连接对象可以是处理器与处理器、处理器与芯片组、芯片组的南北桥、路由器控制芯片等等，属于计算机系统的内部总线范畴。Hyper-Transport技术从规格上讲已经用HT1.0、HT2.0、HT3.0、HT3.1。

QPI总线

主板上一般有两个重要的芯片组，一个称谓北桥，一个称南桥。FSB总线是将CPU连接到北桥芯片的总线。由于FSB不够用，设计上先天不足（这个时候的内存控制器在北桥芯片组内，CPU和内存交换数据总要通过北桥，相当于两个人说话，总要通过第三方，很不方便），因此intel想了个办法，把内存控制器做到了CPU内部，让CPU通过QPI总线直接和内存通讯，不再通过北桥芯片组，这很明显加快了速度。(注意官方名字叫做CSI(Common System Interface公共系统界面)，有的资料中会写CSI)。

QPI是Intel和AMD的HT总线竞争推出的东西。

DMI总线

后来intel发现，CPU通过北桥与显卡相连也很麻烦，不如直接通信方便，这样数据交换更加方便，于是干脆把PCI-E控制器也整合了进了CPU内部，这样一来，相当于北桥芯片组整个都集成到了CPU内部，主板上不再需要这个芯片组了，只剩下了南桥，这时CPU直接与“南桥”相连，他们之间的总线叫做DMI。（北桥的MCH和IOH是逐步集成到CPU内部的）DMI取代了以前的Hub-Link总线。DMI采用点对点的连接方式，具有PCI-E总线的优势。

Intel CPU 经过FSB—>QPI->DMI总线的发展，CPU内部集成了内存控制器和PCI-E控制器，实现了直接和内存及显卡进行数据传输，而由于南桥则整合了几乎所有的I/O功能，因此CPU具备的DMI总线有多高频率意义已经不大了，因为磁盘类设备的速率无法跟上，再高的DMI总线也没有用。

OPI总线

OPI一般用于Intel的低压CPU（U/Y），全称是 On Package DMI interconnect Interface。OPI有两种，2GT/s * 4 lane （最高理论带宽 2 GB/s）和 4GT/s * 8 lane （最高理论带宽 4 GB/s）。Intel笔记本的H和S系列的CPU需要一个分开的“Platform Controller Hub“， U和Y系列的CPU属于MCP（Multi-Chip Packages），也就是说把PCH集成在了CPU里面在H/S系列的CPU中，CPU和PCH的连接是通过DMI的，U/Y使用的是OPI。OPI相对消耗功率更低，但是如果你是2GT/s * 4 lane的OPI， 你可能会影响一部分SSD的性能。
