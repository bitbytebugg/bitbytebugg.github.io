gdb启用/禁用ASLR

为了防范缓冲区溢出攻击等安全漏洞，操作系统会在每次程序启动时随机化进程的地址空间布局，这一机制称为地址空间布局随机化（ASLR）。

在Linux系统中，可以通过 cat /proc/sys/kernel/randomize_va_space 检查当前系统的ASLR是否启用。默认情况下是启用状态（值为2）；若为0则表示被禁用。

之前在使用GDB调试时观察到固定内存地址，是因为GDB默认会禁用ASLR，以便在调试过程中更容易复现问题。你可以通过GDB启用ASLR，使调试环境更接近实际运行环境。启动GDB后，可通过以下命令启用地址空间随机化：

1

(gdb) set disable-randomization off

2.利用signal handler

产生段错误时会产生一个SIGSEGV信号。我们可以添加SIGSEGV的处理函数在程序发生段错误退出之前做一些事情，我们甚至可以使用backtrace(3)函数来保存当前栈内容。

void dump(int signo)
{
        fprintf(stderr,"catch Segmentation fault!!!\n");
#define SIZE 100         FILE *fh;
        if(!(fh = fopen("/tmp/dbg_msg.log", "w+")))
                exit(0);
        void *buffer[100];
        int nptrs;
        nptrs = backtrace(buffer,SIZE);
        backtrace_symbols_fd(buffer, nptrs, fileno(fh));
        fflush(fh);
        exit(-1);
}

int main (void)
{
        signal(SIGSEGV, &dump);
        *((int*)NULL) = 0;

        return 0;
}

定位coredump文件位置的方法：

1

cat /proc/sys/kernel/core_pattern
  • 常见值及其含义：
    • core_pattern值 含义
    • core 当前目录，文件名为 core
    • core.%p 当前目录，文件名包含 PID
    • /var/coredumps/core.%e.%p 指定目录，包含程序名和PID
  • 模式说明符：
    • %p：PID（进程 ID）
    • %u：UID（用户 ID）
    • %g：GID（组 ID）
    • %s：信号编号
    • %t：时间戳
    • %e：可执行文件名
    • %h：主机名

配置coredump文件位置的方法:

1
2

# 设置core_pattern，注意这只是临时修改，重启后失效
echo "/tmp/core.%e.%p" > /proc/sys/kernel/core_pattern

自定义GDB展示样式

主要通过set style命令来实现，该命令的语法结构为：

1

set style TextStyle Attr Value
  • TextStyle：要设置颜色的字符类型
    • source 表示源码
    • address 表示地址
    • filename 表示文件名
    • function 表示函数
    • variable 表示变量
  • Attr 颜色属性：
    • background 背景颜色
    • foreground 字体颜色
    • intensity 字体粗细
  • Value 颜色值：
    • green
    • red
    • yellow
    • bold 加粗

根据文件描述符找到对应的文件路径

1.借助shell查看/proc文件系统

  • p (int)getpid() 获取被调试目标进程的进程ID
  • shell ls -la /proc/{进程ID}/fd/{文件描述符ID} 输出内容->右边的就是该文件描述符对应的文件路径

2.使用call命令调用系统函数

  • p (char*) malloc(256) 分配一块内存用于存储文件路径字符串的值。记住该命令返回的变量${N}
  • call (int) readlink("/proc/self/fd/3", ${N}, 256) 把3替换成要查看的文件描述符ID，该函数可以通过 man 2 readlink 查看具体的帮助信息。该函数的返回值是路径字符串的长度
  • p (char*) ${N} 查看路径字符串的值

# 查看文件描述符对应的文件路径的命令
define showfd
	# 传递给showfd命令的参数个数
	if $argc == 1
		# GDB 中用户自定义变量的命名必须以 $ 开头
		set $inferior_pid = (int)getpid()
		# 借助eval传递$inferior_pid参数，因为直接使用shell命令无法传递gdb里的变量。
		# $arg0为传递给showfd命令的第一个参数
		eval "shell readlink /proc/%d/fd/%d", $inferior_pid, $arg0
	else
		echo "Usage: showfd <fd_number>\n"
	end
end

将上面的代码写入到~/.gdbinit 文件里，启动gdb程序时会自动注册该命令，在gdb环境里通过执行 showfd N 来显示文件描述符N对应的文件路径。

查看全局变量所在的文件

  • set print symbol-filename on ：打开显示符号所在文件的配置开关

调试宏

默认的gcc -g选项等同于-g2，如果需要调试宏信息，需要使用-g3选项。使用-g3选项后在EFL文件中会产生一个.debug_macro的节，该节包含了文件信息、行号信息、以及宏定义具体内容。
