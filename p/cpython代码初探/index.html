下推自动机PDA

理论介绍

Chomsky把文法分为4类，这四类文法从0到3的限制依次增多，表达能力依次减弱：

  • 0型文法：也叫短语文法，能力对应（等价）图灵机。
  • 1型文法：也叫上下文有关文法，能力对应线性有界自动机。
  • 2型文法：也叫上下文无关文法，能力对应下推自动机。
  • 3型文法：正规文法（正则文法），能力对应有限状态自动机（包括NFA和DFA两类，DFA和NFA在计算能力上是等价的，任何可被NFA识别的语言也可以被DFA识别，反之亦然，这种等价性是形式语言理论中的一个重要结论。）

这里状态转移函数就是PDA不断执行的动作，对应算法中的逻辑。解释如下：

  • 其输入有三个维度：
    • 1）当前的状态，Q中的一个元素。
    • 2）接受的0个字符或者1个输入序列中的符号， \( \Sigma \cup {\epsilon} \) 中的一个元素。
    • 3）栈顶的符号， \( \Gamma \) 中的一个元素。所以是这三个维度的叉乘。
  • 根据三个输入执行的动作：修改内部状态和向栈顶压入符号串。

定理：PDA以终态方式接受语言和以空栈方式接受语言是等价的。

算法细节

输入序列的读取指针会根据栈顶符号来决定是否向后移动。栈顶符号的处理依赖栈顶符号的类型来处理。

  1. 栈顶是非终结符A，对应的产生式是BCD，那么我们不会消耗一个输入字符（等价消耗的是空字符），将栈顶的非终结符A弹出，然后按照DCB的顺序将这三个非终结符压入到符号栈里。

  1. 栈顶是非终结符A，对应的产生式是0102B3C，产生式右边的终结符部分和输入序列的字符依次匹配，输入序列读指针依次后移，直到消耗完左边的所有非终结符，然后将栈顶的非终结符A弹出，然后按照顺序C3B压栈。

  1. 总结整个PDA中的转移动作

首先每条边上的转移动作是用一个三元组表示的，例如 \( a, X \rightarrow Y \) ：

  • 元素 \( a\) 表示消耗掉输入序列里的字符，如果用 \( \epsilon \) 表示就是消耗空字符。
  • 元素 \( X\) 表示需要压到符号栈里的元素，可以是终结符也可以是非终结符，如果用 \( \epsilon \) 表示就是说不执行push操作。
  • 元素 \( Y\) 表示需要从符号栈弹出的元素，可以是终结符也可以是非终结符，如果用 \( \epsilon \) 表示就是说不执行pop操作。

四组转换函数的意义：

  1. \( \epsilon, \epsilon \rightarrow Z_0 \) ：Z0表示空栈状态
  2. \( \epsilon, \epsilon \rightarrow S \) ：S是文法的开始符号，在处理输入序列之前会压到栈里。
  3. 中间的这一组操作继续细分：
    1. \( a, a \rightarrow \epsilon \) 多余的，删掉这个表示
    2. \( x, x \rightarrow \epsilon \quad for\quad x \in \Sigma \) ：对于栈顶符号是终结符，且和输入序列符号可以匹配，就弹出栈顶符号并消耗输入字符，不用做压栈操作。
    3. \( \epsilon , A \rightarrow BCD \) ：对于栈顶符号是非终结符，消耗输入序列中的空字符，将栈顶非终结符A弹出，然后按照DCB顺序将右部分符号（终结符和非终结符）依次压栈。
  4. \( \epsilon, Z_0 \rightarrow \epsilon \) ：空栈作为接受状态。

//////  DFA依赖的结构体定义
typedef struct {
    int      lb_type;
    char    *lb_str;
} label;
// arc的标签，记录arc的信息，也就是状态图中有向边的信息。lb_type的取值有两种类型：1.token，也就是token.h中定义的哪些枚举值。lb_str，对lb_type的补充信息，大部分arc的该字段是空，有些lb_type例如token中的NAME表示标识符，需要额外知道标识符的值，所以会有"print", "del", "break"等这些额外的值，这些值就存放在lb_str中。

typedef struct {
    int      ll_nlabels;
    label   *ll_label;
} labellist;

// graminit.c
static label labels[168] = {
  {0, "EMPTY"},
  {1, "def"},
  ...
};
// labels将DFA状态图中的所有有向边上的label信息集中定义在一个数组里。

typedef struct {
    short   a_lbl;      /* Label of this arc */
    short   a_arrow;    /* State where this arc goes to */
} arc;
// 弧，也就是状态图中的有向边包含两部分信息，a_lbl表示arc上的信息，a_arrow表示arc指向的状态序号。注意这里并不是将label结构体直接存储在a_lbl里，而是用一个用于上面labels数组的索引也就是a_lbl字段，如果需要访问arc的标签信息，可以通过labels[a_lbl]来访问。

typedef struct {
    int      s_narcs;  // 有多少条发射出去的有向边，也是下面s_arc数组的长度
    arc     *s_arc;  // 发射出去的有向边的数组
    
    /* Optional accelerators */
    int      s_lower;   /* Lowest label index */
    int      s_upper;   /* Highest label index */
    int     *s_accel;   /* Accelerator */
    int      s_accept;  /* Nonzero for accepting state */
} state;
// 状态定义。

typedef struct {
    int      d_type;    /* Non-terminal this represents */
    char    *d_name;    /* For printing */
    int      d_initial; /* Initial state */
    int      d_nstates;
    state   *d_state;   /* Array of states */
    bitset   d_first;
} dfa;
// DFA状态图结构。
// d_type 指明这个状态图属于哪条文法，用那条文法左边的非终结符的枚举来指明。
// d_name 用来打印信息便于人阅读。
// d_initial 这个DFA的初始状态，每个DFA的状态按照从0开始依次递增编号
// d_nstates 这个DFA中状态的个数，也就是下面d_state数组的长度
// d_state 状态数组
// d_first 这个位字段暂时不清楚作用是什么，好像是计算first集合用的

有了DFA涉及结构体的了解后，接下来详细分析graminit.h/.c中的DFA是怎么样的。主要需要分析dfas[84]这个数组里的信息。首先这个数组长度是84，也就是有84个dfa状态图。可以用下面的程序统计Grammar中也有84个文法产生式，而且每个文法产生式中冒号左边的非终结符都是不同的。从而我们就可以知道Grammar中文法的一些特点：每个非终结符用一条文法来定义（一条文法就是A: B这样的一个文法规则）。每个Grammar中的非终结符对应在graminit.c中变量dfas的一个元素，也就是一个DFA状态图。

fpdef的文法产生式为 fpdef: NAME | '(' fplist ')'。DFA图示如下：

总结以上三个示例的规律： 每个DFA的结束状态上都有一个指向自身的弧，接受的token是ENDMARKER'。ENDMARKER'的字段char *lb_str是"empty"，而表示ENDMARKER(不带单引号)的token的 char *lb_str 字段是0（空指针）。
