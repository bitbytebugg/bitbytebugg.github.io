标准流概念和重定向符号语法

在shell脚本中，默认情况下，总是有三个文件处于打开状态，标准输入、标准输出、标准错误，它们分别对应的文件描述符是0，1，2 。这三个文件描述符常和重定向符号一起使用。

重定向的输入流和输出流/错误流都是指的执行程序的0，1，2这三个文件描述符，而重定向符号 > 或者 < 的左边是一个执行的程序（可能带着参数），右边是一个文件。

输出流（包括错误流）的重定向符号是 >:

  • > 是将左边的输出流重定向到右边文件
  • >> 将左边的输出流以追加的方式重定向到右边
  • 1> 左边的输出流重定向右边，和 > 等效，中间不能有空格
  • 2> 左边的错误流重定向到右边，中间不能有空格

输出流（包括错误流）合并重定向 的操作是&>和2>&1，它们都是将标准输出和标准错误合并重定向到一个目标，但是语法和行为上有一些细微区别：

  • 前者可以直接合并输出和错误到右边的文件；后者需要结合 > 使用，稍显复杂。不结合使用时只合并左边程序的输出和错误，右边不能接一个文件。总之就是前者用法是直接合并流到文件，而后者自身单独使用表示只合并流。
    • 例如ls file &> output.log等价于ls file > output.log 2>&1。而 ls file 2>&1 单独使用只表示将错误流合并到输出流，不会重定向到某个文件。
    • 2>&1 的使用要注意顺序，要么紧跟在某个命令后面，要么放在命令重定向之后。
  • 前者是 Bash扩展语法，可能不兼容某些shell终端；后者是POSIX标准语法，兼容性更好。

输入流的重定向符号是 <

  • < 将右边的文件重定向到左边程序的输入流

  • << 这个符号也叫做 Here Document, 在Shell中的含义是允许在脚本中直接嵌入输入数据，直到遇到指定的“分界符”为止。

    • 用法是 [命令] << 分界符：
      • 左边的命令部分可以省略，如果左边的命令省略，用法相当于Shell脚本中的多行注释
      • 当左边命令存在时，会将第一遇到分界符到第二次遇到分界符之间的内容作为多行文本传入左边命令的输入流

  • <<< 这个符号也叫做 Here String，在Shell中的含义是将一个字符串作为标准输入传递给命令。

    • 用法：命令 <<< "字符串"

  • 对比Here Document和Here String：

    • 前者适合传入通过键盘键入的多行文本
    • 后者适合传入通过键盘键入的单行文本

特殊的重定向设备文件 /dev/null

/dev/null 是一个特殊的文件，写入到它的内容都会被丢弃；如果尝试从该文件读取内容，那么什么也读不到。/dev/null 这个文件非常有用，将命令的输出重定向到它，会起到"禁止输出"的效果。 如果希望屏蔽 stdout 和 stderr，可以这样写：command > /dev/null 2>&1

特殊的重定向设备文件 /dev/zero

/dev/zero 也是一个特殊的文件，它是一个只读的零字节流源，它具有如下特点：

  • 当你从 /dev/zero 读取时，它会无限地提供空字符（ASCII 码为 0 的字符，即 \0）
  • 向 /dev/zero 写入的任何数据都会被静默丢弃
