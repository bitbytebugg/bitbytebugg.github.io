操作数类型标记

x86 指令集不完全是严格的二地址指令集，而是以二地址指令为主，同时包含部分单地址、零地址甚至三地址指令，具有混合特性。但是多数指令是二地址形式，也就是说会包含两个操作数。

x86指令手册中用一个大写字母和一个小写字母来表示操作数的类型：

  • 第一个大写字母含义 （也叫做 Operand Addressing Method Codes）：
    • E：表示操作数可以是寄存器或内存地址（Effective address）。
    • G：表示操作数必须是通用寄存器（General-purpose register）。
    • I：表示立即数（Immediate value）。
    • M：表示操作数必须是内存地址（Memory operand）。
  • 第二个小写字母含义 （也叫做 Operand Type Codes）：
    • b：字节（8位，如 AL, BL）。
    • w：字（16位，如 AX, BX）。
    • d：双字（32位，如 EAX, EBX）。
    • q：四字（64位，如 RAX, RBX）。
    • v：可变大小，由当前运行模式决定（16/32/64位模式，如 AX、EAX 或 RAX）。

更多操作数类型标记含义参考 x86 Instruction Decoding - Appendix A: Opcode Map 和 80386 PRM

Part 1: Prefix

prefix如果有的话就是指令最开始的部分。prefix部分不是必须的，最少0字节，最多4个字节。

32位x86指令中的前缀也被称为legacy prefix，AMD在扩展的x86-64指令中新增的前缀称为 REX 前缀。前缀机制是x86指令集向后兼容性的重要部分，允许新特性在不破坏旧代码的情况下被添加。

legacy prefix概述

legacy prefix 的作用：

  • 调整内存操作数的属性
  • 增强指令的功能
  • 提供额外的作用

leagcy prefix 的分类：

  • operand size override prefix：66H — 改变默认操作数大小
  • address size override preifx：67H — 改变默认操作数地址大小
  • segment override prefix：改变 memory 操作数段选择子，包括：
    • 2E — CS register
    • 3E — DS register
    • 26 — ES register
    • 64 — FS register
    • 65 — GS register
    • 36 — SS register
  • rep/repz prefix：F3H — 串指字重复执行
  • repnz prefix：F2H — 串指字重复执行
  • lock prefix： F0H — LOCK

REX prefix概述

REX prefix的作用：

  • 访问 64 位的资源：使用 64 位的操作数，改变指令操作数的 default operand size（可以使用 64 位 operands），同样是起 operand size override 的作用
  • 访问 x64 体系扩展资源：x64 体系新增的 register，如：r8 - r15，xmm8 - xmm15 等

Part 3: ModRM

ModRM 最主要作用是对指令的 operands 提供寻址，另一个作用是对 Opcode 进行补充，而 SIB 则是对 ModRM 进行补充寻址。

有两种情况下是无需用 ModRM 提供寻址的：

  1. 一部分操作数是寄存器的，它直接嵌入 Opcode 中。
  2. 一部分操作数是立即数的，它直接嵌入指令编码中。

ModRM的结构(整体1字节)分为三部分：

  1. ModRM.mod 7-6位提供寻址模式:
    • 11 = register
    • 00, 01, 10 = memory
  2. ModRM.reg 5-3位提供 register 或者对 Opcode 进行补充
  3. ModRM.r/m 2-0位提供 register 或者 memory 依赖于 ModRM.mod

ModRM.mod

2 位组成 4 种寻址模式，总的来说，只有两种寻址模式，就是内存寻址模式和寄存器寻址模式。

  • 00 提供 [base] 形式的 memory 寻址
  • 01 提供 [base + disp8] 形式的 memory 寻址
  • 10 提供 [base + disp32] 形式的 memory 寻址
  • 11 提供 register 寻址。

ModRM.reg

3 位组成 8 个寄存器 ID 值，从 000 ~ 111，对应于 EAX、ECX、EDX、EBX、ESP、EBP、ESI 以及 EDI 寄存器。 这个 ID 值可以被 REX.R 扩展为 4 位： 0000 ~ 1111（对应于 RAX 到 R15 寄存器）。

ModRM.r/m

r/m 意即：registers/memory，提供对 register 或 memory 的寻址，它与 ModRM.mod 是密切相关的。

Part 4: SIB

SIB 是对 ModRM 寻址的一个补充：

  • ModRM 提供的是 registers 寻址、[register] 寻址（寄存器间接寻址）以及 [register + displacement]（寄存器基址寻址）。
  • SIB 提供的是 [base + index * scale] 这种形式的寻址。即：基址 + 变址 同样，SIB 是可选的，前面已经介绍：SIB 字节由 ModRM.r/m = 100 引导出来，指令中命名用了 [base + index] 这种地址形式时，必须使用 SIB 进行编码。

SIB结构：

  • SIB.scale 7-6位，提供 index 寄存器乘数因子 scale
    • 00 乘数因子位 2^0 = 1
    • 01 乘数因子位 2^1 = 2
    • 10 乘数因子位 2^2 = 4
    • 11 乘数因子位 2^3 = 8
  • SIB.index 5-3位，提供 index 寄存器寻址
  • SIB.base 2-0位，提供 base 寄存器寻址
